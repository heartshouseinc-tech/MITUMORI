## 1. 導入・課題提起

皆さん、お疲れ様です。本日は少しお時間をいただいて、見積書作成の業務について、新しい取り組みをご紹介させてください。

今、スズホームでは見積書を作る際、現場で聞いた内容をメモや記憶に頼って持ち帰り、事務所でExcelに手入力する、という流れになっています。この方法にはいくつか困りごとがあると感じています。

一つ目は、担当者の記憶や経験に頼る部分が大きいことです。品名や仕様、数量を後から思い出しながら入力するので、聞き漏らしや入力ミスが起きやすくなります。

二つ目は、現場と事務所で二度手間になっていることです。現場で一度聞いた内容を、事務所でもう一度Excelに整理し直す作業が発生しています。

三つ目は、過去の案件が一覧で見えないことです。今どの案件が進行中で、どれが送付済みで、どれが受注できたのか、会社として把握する仕組みがありませんでした。

こうした課題を解決するために、スマートフォンとAIを使った見積作成の仕組みを作りました。これから、その内容をご説明します。

## 2. 解決策の全体像

今回作ったのは、「現場ではスマートフォン、事務所ではパソコン」という、二段構えのワークフローです。全体の流れを説明します。

まず現場では、スマートフォンで音声を吹き込むか、写真を撮るか、あるいは直接テキストを入力します。これをAIが自動で読み取り、品名・仕様・数量・単位といった見積項目に分解してくれます。

次に、単価の決め方を選べるようにしました。現場でその場で単価を確定させる「現場確定」、パソコンでの仕上げ時にまとめて決める「PC後決め」、そしてAIにおおよその相場を仮に見積もらせる「AI仮単価」の3つのモードです。案件の性質に応じて使い分けられます。

作った見積は「案件」として保存され、消えることはありません。案件ごとに「下書き」「PC仕上げ待ち」「送付済み」「受注」「失注」といったステータスが付き、これを一覧のボード形式で会社全体がいつでも見渡せるようになっています。誰がどの案件を今どこまで進めているか、一目で分かります。

そして事務所では、パソコンから同じ案件を開いて仕上げ作業ができます。仕上げ方法は二通り用意しました。一つは、御社の決まったExcel様式にそのまま数値を書き出す方法。もう一つは、Excel操作をアシストするツールに貼り付けるための指示文をコピーする方法です。どちらのやり方にも対応できます。

最後に、見積の粗利（原価に対してどれくらいの利益が乗っているか）をその場で確認できるようにしました。金額を入れるだけでなく、その案件が採算に合っているかどうかも同時に見える仕組みです。

## 3. 具体的な機能・使い方の説明

実際の画面の流れに沿って、もう少し具体的にご説明します。

アプリを開くと、まず案件の一覧画面が出てきます。これがさきほど申し上げたボード形式の画面で、進行中の案件がステータスごとに並んでいます。新しい案件を作るときは、ここから「新規案件を作成」のボタンを押すだけです。

現場では、まず単価の決め方（現場確定・PC後決め・AI仮単価のどれか)を選びます。そのあと、音声で喋る、写真を撮る、あるいはテキストで打ち込む、いずれかの方法で依頼内容を入力します。AIが自動でこれを見積項目に分解してくれるので、手作業で一つずつ入力する必要はありません。品名や仕様が多少あいまいな喋り方でも、AIが解釈して形にしてくれます。

単価が必要な項目については、登録してあるメーカーの単価表や自社の歩掛かり表と自動で照合し、金額を当てはめます。マッチしなかった項目は、AIがおおよその相場を提示してくれるので、それを見て現場で微調整することもできます。

事務所側の担当者は、パソコンから同じ案件を開くと、現場で入力された内容がそのまま表示されます。ここで正式なExcelの様式に仕上げます。仕上げ方法は、決まったExcel様式にワンクリックでそのまま書き出す方法と、Excel操作をアシストするツールに貼り付ける指示文をコピーする方法の、どちらか都合の良い方を選べます。どんな列構成のExcel様式にも対応できる作りになっているので、複数の様式を使い分けている場合でも安心して使えます。

そして最後に、見積の合計金額の下に、原価の小計と粗利、粗利率が表示されます。想定より粗利が低い案件は、色を変えて警告するようにしていますので、金額を入れて終わりではなく、その場で採算を確認しながら見積を作ることができます。

## 4. 現在の進捗（正直にお伝えします）

ここまでご説明した機能は、すべてプログラムとしては完成しており、実際にアプリの本番環境に組み込み済みです。ただ、正直にお伝えしなければならない点がいくつかあります。

まず、実際のスマートフォンとパソコンを使った、本番のFirebase環境での一連の動作確認は、まだ行えていません。開発環境の制約で、外部のFirebaseサーバーに接続しての通し確認ができず、プログラムの構文チェックと、ダミーデータを使った画面表示の確認までしか行えていない状態です。ですので、これから皆さんに実際に触っていただく中で、思わぬ不具合が見つかる可能性があります。

次に、単価表への原価データの入力が、まだ済んでいません。メーカーの単価表や自社の歩掛かり表には、売値は登録されていますが、原価がまだ入力されていないため、現状では粗利チェックの欄が「原価未設定」と表示されたままになっています。実際に粗利を確認できるようにするには、このデータの入力作業にご協力いただく必要があります。

また、粗利が低いと警告する基準を、今は仮に20%と設定していますが、これは暫定の数値です。実際の案件データを見ながら、御社の実情に合った数値に調整していく必要があります。

最後に、ログインなしで使える簡易版と、他社に配布するための白紙のひな形については、今回の新しい機能にはまだ対応していません。ただ、これは社内での通常業務には影響しない部分です。

## 5. 導入のお願い・次のアクション

以上を踏まえて、皆さんに4つお願いしたいことがあります。

一つ目は、まず数件、実際の案件でこの仕組みを試験的に使っていただきたいということです。実運用の中でしか見つからない不具合や使いにくさがあると思いますので、率直なフィードバックをいただければと思います。

二つ目は、単価表への原価データの入力にご協力いただきたいということです。これがないと粗利チェックの機能が活きてきませんので、まずは主要な項目だけでも入力を進めさせてください。

三つ目は、粗利の警告基準（今は仮に20%としています）について、実際の案件を見ながらご意見をいただきたいということです。

四つ目は、運用上の注意点です。事務所の担当者がパソコンで他の人が作った案件を仕上げる、という使い方をする場合、そのアカウントには管理者権限を付与しておく必要があります。これは案件のデータを、作った本人か管理者しか編集できない仕組みにしているためです。運用の準備の際にご相談させてください。

私からの説明は以上です。ご不明な点や気になる点がありましたら、ぜひご質問ください。
